# Demo Guide

推荐演示顺序控制在 5-8 分钟，先展示稳定缓存案例，再展示真实查询开关。

## 1. 首页

说明项目定位：面向供应商准入场景的 AI 尽调与风险研判系统。重点展示三步流程：企业输入、联网搜索/证据抽取、规则评分/报告生成。

## 2. 缓存 Demo 案例

进入 `/demo`，依次打开：

1. 比亚迪股份有限公司：低风险案例，重点看真实 URL、普通记录不参与评分。
2. 华为技术有限公司：高关注主体，重点看人工复核与政策提示。
3. 小米通讯技术有限公司：标准准入，重点看企业画像补全。
4. 高风险供应商案例：展示规则命中和 high 风险报告。

每个案例都应检查：

- task_id
- 搜索 query 数
- web_search_results 与真实 URL
- 企业画像字段
- 可评分风险证据
- Markdown/PDF 下载

## 3. 历史任务

进入 `/tasks`，演示按 task_id 打开、任务列表排序、报告导出按钮绑定当前行 task_id。

## 4. 真实查询模式

进入新建任务页面，输入企业名称和采购金额。真实查询会消耗腾讯云与 LLM API，并且搜索结果可能随时间波动。

系统控制：

- Provider 未配置时按钮置灰。
- 全站每日默认最多 20 次真实查询。
- 同一企业 7 天内已有 completed 任务且存在搜索结果时优先复用。

## 5. 报告说明

报告中的分数和等级由规则引擎生成，LLM 只优化表达，不改变证据、URL、分数、等级和准入建议。
